■ 문서 목적

본 문서는 다음 목적을 가진다.

1. 프로그램의 “무엇을 만들 것인가”를 명확히 정의한다.
2. 사용자 요구사항을 구조화된 스펙으로 정리한다.
3. 상태 / 권한 / 예외 / 테스트를 사전에 통제한다.
4. CODE 생성 가능한 수준까지 구체화한다.

본 문서는 구현 방법이 아닌 기능 정의 문서이다.
구현 방식은 반드시 CM_ABAP패턴.md를 따른다.

14. CODE 생성 기준

- 모든 F-xxx는 FORM으로 매핑
- 모든 S-xxx는 status_check 로직으로 매핑
- 모든 A-xxx는 authority_check 로직으로 매핑
- SPEC에 없는 DB 변경 금지
- SPEC에 없는 상태 전이 금지
- SPEC 확정 상태에서만 CODE 생성 허용
