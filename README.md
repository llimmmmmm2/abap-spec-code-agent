# 🚀 ABAP Spec/Code 생성 에이전트 (abap-spec-code-agent)

> **요구사항 정의부터 ABAP 코드 생성까지 지원하는 AI 기반 개발 보조 에이전트**

본 프로그램은 SAP ABAP 개발 과정에서 요구사항을 구조화하여 **SPEC(설계서)**와 **CODE 초안**을 생성하는 도구입니다. 요구사항 입력 → SPEC 생성 → SPEC 확정 → CODE 생성 과정을 단계적으로 지원하며, 생성 결과와 사용자 로그를 기록하여 향후 규칙 개선에 활용할 수 있도록 설계되었습니다.

---

## 📝 1. 개발 배경
최근 AI 기술이 개발 업무에 도입되면서 단순 코드 작성 작업은 점차 자동화될 가능성이 높아지고 있습니다. 이에 따라 운영 인력 역시 단순 코더보다는 요구사항을 정리하고 설계를 구조화하는 업무에 더 집중할 필요가 있습니다.

작년에는 '프롬ABAP'을 활용하여 코드 생성 가능성을 테스트하였으나, SPEC 작성 과정이 자동화되지 않아 번거로움이 있었습니다. 본 에이전트는 이러한 경험을 바탕으로 **요구사항을 코드 구현이 가능한 수준의 SPEC으로 구조화**하여 이 간극을 해결합니다.

---

## 🎯 2. 대상 사용자
본 프로그램은 완전한 비개발자보다는 SAP 환경에 대한 이해가 있는 사용자를 대상으로 합니다.
**SAP 운영 인력 및 업무 담당자**
**ABAP 개발 경험이 많지 않은 사용자**
**SAP 테이블 / 구조 / 필드 개념**을 이해하고 있는 사용자

---

## ✨ 3. 주요 기능
**요구사항 구조화 입력**: 기본 정보, 업무 개요, 데이터 정의, 화면 구성, 상태/권한, 범위 정의 등 항목별 가이드 입력
**SPEC 초안 자동 생성**: 입력된 요구사항을 기반으로 논리적인 설계서 초안 작성
**미확정 항목 보완**: AI가 분석한 불확실하거나 누락된 항목을 사용자 입력을 통해 보완
**최종 SPEC 생성 및 다운로드**: 모든 정보가 반영된 최종 SPEC을 생성하고 문서화
**ABAP CODE 초안 생성**: 확정된 SPEC을 분석하여 실행 가능한 ABAP 코드 생성 및 다운로드
**사용자 로그 및 피드백 저장**: 지속적인 규칙 개선을 위한 데이터 축적 및 분석

## 📋 6. 화면 및 프로세스 상세 안내

### STEP 1: 요구사항 입력
사용자가 프로그램 요구사항을 다음 6가지 항목으로 나누어 상세히 입력합니다.
**기본 정보:** 프로그램 명, T-Code, 개발 유형 등
**업무 개요:** 비즈니스 프로세스 및 개발 목적
**데이터 정의:** 참조 테이블, Internal Table 구조, 주요 필드 정보
**화면 구성:** Selection Screen 필드, 조회 조건, 버튼 레이아웃
**상태 / 권한:** UI Status(GUI 타이틀, 버튼), 권한 체크(Authority Check)
**범위 정의:** 처리 로직의 시작과 끝, 제외 조건 등

### STEP 2: SPEC 초안 생성
입력된 요구사항을 AI가 분석하여 가독성 있는 마크다운 형태의 SPEC 초안을 생성합니다.

### STEP 3: 미확정 항목 보완
SPEC 초안에서 로직이 모호하거나 데이터 정의가 누락된 부분을 AI가 찾아내어 사용자에게 질문을 던집니다. 사용자의 보완 입력을 통해 설계의 완성도를 높입니다.

### STEP 4: 최종 SPEC 생성
보완된 모든 정보를 기반으로 최종 SPEC을 확정합니다. 확정된 SPEC은 파일로 다운로드하여 문서화할 수 있습니다.

### STEP 5: ABAP CODE 생성
최종 확정된 SPEC과 rules/ 폴더에 정의된 ABAP 표준 패턴을 결합하여 가독성 높고 구문 오류가 적은 ABAP 코드를 생성합니다.

## 📅 7. 향후 계획
**코드 생성 안정화:** 구문 오류 최소화 및 복잡한 로직(Join, Parallel Processing 등) 처리 개선
